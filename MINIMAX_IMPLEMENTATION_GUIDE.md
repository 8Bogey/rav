# 🤖 Minimax Agent Implementation Guide: Drift + Convex Local-First (Deep Dive)

**Attention Agent:** You are the primary executor migrating the `mawlid_al_dhaki` app from a relational, cloud-dependent architecture (Drift + Supabase) to a modern, offline-optimized Local-First architecture using **Drift (Local) + Convex (Cloud)**.

This guide acts as your ultimate tech spec. Read the code patterns provided below; they cover ~20% of the actual boilerplate needed. **You must understand the exact goal of each phase before running `bd ready`.**

---

## 🛑 Absolute Rules for the Agent
1. **Never Edit the UI/UX visually:** The app uses a specific "Bitepoint POS" design system. You are ONLY changing Riverpod data providers. DO NOT change padding, colors, or visual layouts.
2. **Drift remains the Local DB:** We are NOT using Isar due to dependency conflicts. We are keeping Drift (SQLite).
3. **Global String IDs (UUID):** Change all primary keys in Drift (`IntColumn`) to `TextColumn`. The IDs will be the unique string `_id` generated by Convex. Never use local auto-incrementing integers.
4. **Soft Delete Only:** Every Drift table and Convex document must have an `isDeleted` boolean. Never use `DELETE` operations.
5. **Tenant & Role Isolation:** Every Convex query/mutation and Drift table MUST have an `ownerId`.

### ☁️ [EPIC 2] Convex Backend Schema & Server Functions
**Goal:** Define the Multi-Tenant Cloud Schema in TypeScript.

**What to do:**
1. Port all 7 SQL tables into Document Schemas in `convex/schema.ts`.
2. Write Mutations that strictly enforce LWW (version numbers) and `ownerId`.

**Schema Boilerplate (`convex/schema.ts`):**
```typescript
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  subscribers: defineTable({
    ownerId: v.string(), // Multi-tenant isolating key
    name: v.string(),
    code: v.string(),
    cabinet: v.string(),
    status: v.number(),
    accumulatedDebt: v.number(),
    
    version: v.number(), // For Last-Write-Wins checks
    isDeleted: v.boolean(), // Soft delete flag
  }).index("by_ownerId", ["ownerId"])
    .index("by_code", ["code"]),
});
```

**Mutation Boilerplate (`convex/mutations/subscribers.ts`):**
```typescript
import { mutation } from "../_generated/server";
import { v } from "convex/values";

export const saveSubscriber = mutation({
  args: { 
    id: v.optional(v.id("subscribers")), 
    version: v.number(),
    ownerId: v.string(),
    name: v.string(),
    // ...other args
  },
  handler: async (ctx, args) => {
    // 1. Authenticate Request
    const identity = await ctx.auth.getUserIdentity();
    if (!identity) throw new Error("Unauthenticated");

    // 2. Tenant Isolation
    if (args.ownerId !== identity.subject) throw new Error("Unauthorized");

    if (args.id) {
      // 3. Update Existing (LWW Check)
      const existing = await ctx.db.get(args.id);
      if (!existing || existing.ownerId !== identity.subject) throw new Error("Unauthorized");
      
      if (args.version <= existing.version) {
         return { success: false, reason: "stale" }; // Reject old data
      }
      const { id, ...data } = args;
      await ctx.db.patch(id, data);
      return { success: true, id };
    } else {
      // 4. Create New
      const { id, ...data } = args;
      const newId = await ctx.db.insert("subscribers", data);
      return { success: true, id: newId };
    }
  },
});
```

### 💾 [EPIC 3] Drift Local Database Layer
**Goal:** Restructure the old SQL tables to accept UUIDs and Outbox tasks.

**What to do:**
1. Edit `lib/core/database/tables/`. Change `IntColumn get id => integer().autoIncrement()()` to `TextColumn get id => text()()`.
2. Add `version`, `isDeleted`, `updatedAt`, `ownerId`.
3. Create the `OutboxTable` for offline writes.

**Outbox Table Boilerplate (`lib/core/database/tables/outbox_table.dart`):**
```dart
import 'package:drift/drift.dart';

class OutboxTable extends Table {
  TextColumn get id => text()(); // UUID entry
  TextColumn get tableName => text()(); // e.g., 'subscribers'
  TextColumn get operationType => text()(); // 'create', 'update', 'delete'
  TextColumn get documentId => text()(); // The target drift row ID
  TextColumn get payload => text()(); // JSON string of the row
  DateTimeColumn get createdAt => dateTime()();
  IntColumn get retryCount => integer().withDefault(const Constant(0))();
  TextColumn get status => text().withDefault(const Constant('pending'))(); // pending, failed, synced

  @override
  Set<Column> get primaryKey => {id};
}
```

### 🌉 [EPIC 4] Sync Bridge (Drift <-> Convex)
**Goal:** Implement the engine that syncs offline edits and pulls new data.

**What to do:**
1. `OutboxProcessor`: Reads Drift `OutboxTable` where status is 'pending', sends mutation to Convex. If success, marks 'synced'.
2. `SubscriptionListener`: Subscribes to Convex `queries.ts`, applies changes locally via `insertOnConflictUpdate`.

**Outbox Write Logic (Used inside DAOs):**
```dart
// Example of recording an offline payment
Future<void> savePayment(Payment payment) async {
  await transaction(() async {
    // 1. Update timestamp & version
    final updatedPayment = payment.copyWith(
      updatedAt: DateTime.now(),
      version: payment.version + 1,
    );
    
    // 2. Save explicitly to local db
    await into(paymentsTable).insertOnConflictUpdate(updatedPayment);
    
    // 3. Queue the network task
    final outboxEntry = OutboxCompanion.insert(
      id: const Uuid().v4(),
      tableName: 'payments',
      operationType: 'update',
      documentId: payment.id,
      payload: jsonEncode(updatedPayment.toJson()),
      status: 'pending',
      createdAt: DateTime.now(),
    );
      
    await into(outboxTable).insert(outboxEntry);
  });
}
```

### 🔄 [EPIC 6] Service Layer & Provider Refactoring
**Goal:** Connect UI to Drift using streams, completely filtering out soft-deleted items.

**What to do:**
Rewrite DAOs so the UI only gets fresh, undeleted data.

**DAO Query Boilerplate:**
```dart
Stream<List<Subscriber>> watchActiveSubscribers(String tenantId) {
  // MUST filter by ownerId and isDeleted == false!
  final query = select(subscribersTable)
    ..where((t) => t.ownerId.equals(tenantId))
    ..where((t) => t.isDeleted.equals(false));
    
  return query.watch();
}
```

**Riverpod Boilerplate:**

### 🔐 [EPIC 7] Auth & Role-Based Access Control (RBAC)
**Goal:** Differentiate between Admins and Workers (Collectors).

**What to do:**
1. Add an `assignedWorkerId` to field operations (like Cabinets).
2. If `user.role == 'admin'`, they query all tenant data.
3. If `user.role == 'worker'`, the app injects an extra `where()` clause into Convex Subscriptions and Drift Queries.

**Implementation Example:**
```dart
Stream<List<Cabinet>> getCabinetsForUser(User user) {
  final query = select(cabinetsTable)
    ..where((t) => t.ownerId.equals(user.tenantId))
    ..where((t) => t.isDeleted.equals(false));

  if (user.role == 'worker') {
    // Workers only see their assigned cabinets
    query.where((t) => t.assignedWorkerId.equals(user.id));
  }
  
  return query.watch();
}
```
